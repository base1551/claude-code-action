# Claude OAuth セキュリティチェックリスト

## 🔒 導入前セキュリティチェック

### [ ] 基本要件
- [ ] リポジトリがプライベートに設定されている
- [ ] Claude Max サブスクリプションが有効
- [ ] GitHub Actionsが有効化されている
- [ ] 管理者権限を持つユーザーのみがSecretsアクセス可能

### [ ] 権限設定
- [ ] ワークフローの `permissions` が最小権限に設定
- [ ] `secrets: write` のみが必要な権限として設定
- [ ] `actions: write` 権限が不要であることを確認
- [ ] 外部コントリビューターのPR実行が制限されている

### [ ] Secrets設定
- [ ] `CLAUDE_ACCESS_TOKEN` が正しく設定されている
- [ ] `CLAUDE_REFRESH_TOKEN` が正しく設定されている  
- [ ] `CLAUDE_EXPIRES_AT` が正しい形式で設定されている
- [ ] 不要なSecretsが設定されていない

## 🛡️ 実装セキュリティチェック

### [ ] コード検証
- [ ] トークンがログに平文で出力されていない
- [ ] エラーメッセージにセンシティブ情報が含まれていない
- [ ] 適切な暗号化が実装されている（sodium推奨）
- [ ] 入力値のバリデーションが実装されている

### [ ] API呼び出し
- [ ] HTTPS通信のみを使用
- [ ] 適切なUser-Agentが設定されている
- [ ] レート制限が実装されている
- [ ] タイムアウト設定が適切

### [ ] エラーハンドリング
- [ ] 機密情報がエラーログに含まれていない
- [ ] 適切なフォールバック処理が実装されている
- [ ] 失敗時の通知が適切に設定されている

- [ ] ログ保持期間の設定

- [ ] 管理者権限の最小化

### [ ] 定期メンテナンス
